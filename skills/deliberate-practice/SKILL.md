# 刻意练习方法论 · Deliberate Practice

> "以大多数人的努力程度之低，根本轮不到拼天赋。"

刻意练习是专门为了"提高能力"而设计出来的练习方法。底层信念：高手是练出来的，不是天生的。

## 体系导航

本 Skill 是「科学成事」体系的行动层，与另外两个 Skill 构成闭环：

| 你要做什么 | 用哪个 Skill | 定位 |
|:---|:---|:---|
| 理解什么叫科学地做事 | **ymodel** | 世界观层 |
| 定位能力阶段、诊断瓶颈 | **competence-reflection** | 诊断层 |
| 设计具体练习方案 | **deliberate-practice**（本Skill） | 行动层 |

### 前置检查

开始设计练习方案前，确认以下条件是否满足：

| 检查项 | 不满足时 | 满足后 |
|:---|:---|:---|
| 用户是否知道自己想提升的具体能力？ | 先追问到具体能力维度，不要接受"提升沟通能力"这种模糊目标 | → 进入 Step 1 |
| 用户是否知道自己的瓶颈在哪？ | 建议先用 **competence-reflection** 做诊断，拿到瓶颈定位再来 | → 进入 Step 2 |
| 用户的方向是否正确？ | 如果用户不确定自己在练什么、为什么练，建议先用 **ymodel** 四步法做快速自检 | → 进入 Step 2 |

> 练习方案建立在正确的自我定位之上。**方向错了，练得越猛偏离越远。**

## 核心框架：ABCD 四要素

四个要素缺一效果就会大打折扣。诊断任何练习问题，先逐项检查 ABCD：

### A —— 固定套路（Routine / Framework）

> "但凡重复，必有套路；无套路，不练习。"

必须有稳定的执行框架——方法论、SOP、检查清单、模型。类比：围棋的定式、编程的算法、数学的公式。

没有稳定套路的随机练习，很容易在新手阶段就停滞不前。

**策略分级**：
- **基础**：创建文档——给练习建立一个指导文档
- **进阶**：使用现成模型、划分阶段模型
- **高阶**：提炼底层模型、自建模型

**操作指引**：先问用户"你现在练习有固定的框架或流程吗？"如果没有，帮他们找一个或建一个。

### B —— 非舒适区（Beyond Comfort Zone）

> "保持非舒适区最简单的方法——去学习和追求最佳实践。"

不能习惯性简单重复，要挑战自己不会做的事。对标最佳实践，保持渴望，努力靠近高水平。

**策略分级**：
- **基础**：严格执行流程和 Checklist，不打折扣
- **进阶**：能力拉满、对标最佳、每次提升 20%
- **高阶**：永远 60 分心态、构建"想象中的最佳实践"

**操作指引**：问用户"你上次练习时，有没有哪一步觉得特别吃力？"如果全部轻松，说明不在非舒适区。

### C —— 及时反馈（Timely Feedback）

> "及时修正姿势，避免在低效/错误的道路上一路狂奔。"

练习后需要知道做得好不好，差距在哪里。没有反馈的练习可能越练越偏。

**策略分级**：
- **基础**：自我反馈、事实反馈（记录数据）
- **进阶**：专家反馈、主动索取、版本对比、同伴反馈
- **高阶**：建立最佳实践池子、审美雷达图

**操作指引**：帮用户设计反馈机制——谁能给反馈？用什么标准？多久一次？

### D —— 大量重复（Deliberate Repetition）

> "10 倍重复 = 10 倍进步。"

规模化增加练习频次和循环次数。重复是为了具体的实操能力，听课/记笔记不算重复练习。

**策略分级**：
- **基础**：日常固定练习（固定日期、场景、项目节点）
- **进阶**：专项练习（化整为零）、主动找机会练习
- **高阶**：叠加暗时间、迁移练习（升级底层模型）

**操作指引**：帮用户算清楚"一周能练几次"并设计具体排期。

## 工作流：用 ABCD 诊断和设计练习

当用户想提升某项能力时，按以下流程操作：

### Step 1：明确目标能力

确认用户想提升的具体能力。越具体越好：

- 好的目标："提升用户访谈中追问深层次需求的能力"
- 差的目标："提升沟通能力"

### Step 2：ABCD 四项诊断

对每一项打分（1-5 分）并给出判断：

| 要素 | 诊断问题 | 常见问题 |
|:---:|:---|:---|
| **A 套路** | 有没有固定的练习框架/模型/SOP？ | 随机练习，每次做法不同 |
| **B 非舒适区** | 练习时是否感到"吃力但有收获"？ | 一直在舒适区重复擅长的事 |
| **C 反馈** | 练习后是否能及时知道差距？ | 练完就完了，没有复盘和反馈 |
| **D 重复** | 练习频次是否足够？ | 频次太低，间隔太久 |

**瓶颈判断**：得分最低的 1-2 项就是核心瓶颈，优先解决。

### Step 3：设计练习方案

针对瓶颈，从策略武器库中挑选对应级别的策略，输出一份可执行的练习方案。

方案格式：

```
## [能力名称] 刻意练习方案

### 目标
[具体的、可衡量的目标]

### ABCD 现状诊断
- A（套路）：[分数]/5 — [现状描述]
- B（非舒适区）：[分数]/5 — [现状描述]
- C（反馈）：[分数]/5 — [现状描述]
- D（重复）：[分数]/5 — [现状描述]

### 核心瓶颈
[得分最低的 1-2 项]

### 练习设计
#### A 套路
[具体的框架/模型/SOP]

#### B 非舒适区
[具体的挑战设计]

#### C 反馈机制
[反馈来源、频率、标准]

#### D 重复计划
[频次、时间段、排期]

### 阶段目标
- 2 周后：[里程碑]
- 1 个月后：[里程碑]
- 3 个月后：[里程碑]
```

### Step 4：动机对齐（可选但推荐）

确认用户的练习动机是否足够支撑长期坚持：

- **最强动机**：人生红点（长期价值追求）+ 必经之路（达成红点必需的能力）
- **有效动机**：对标榜样、渴望成为高手、年度晋级、阶段目标
- **危险信号**：只有外部压力，没有内在驱动力——需要帮用户找到内在意义

## 个人版 vs 团队版

刻意练习的底层逻辑从个人到团队完全一致：

| 维度 | 个人版（刻意练习） | 团队版（苦练基本功） |
|:---:|:---|:---|
| A | 套路模板 | 方法模型 |
| B | 非舒适区 | 最佳实践 |
| C | 及时反馈 | 实事求是 |
| D | 大量重复 | 学习制度 |

当用户提到团队/组织的能力建设时，自动切换到团队版框架。

## 经典案例参考（用于类比启发）

当用户说"不知道怎么练"或"ABCD 听得懂但不知道怎么落地"时，引用案例做类比。

### 案例 1：AI 提示词练习（Truman，2023 年至今）

目标：提示词从 30 分到 80 分。核心套路："魂 + 工具 + 迭代"模型。

| ABCD | 具体做法 |
|:---:|:---|
| A 套路 | 提炼"魂（角色）+ 工具（框架）+ 迭代（追问修正）"模型，每次用固定模板写提示词 |
| B 非舒适区 | 对标 OpenAI 官方指南和社区大神的写法，每次多考虑一个维度 |
| C 反馈 | 看 AI 输出质量（事实反馈）+ 保存每次版本做对比 + 参考获奖者写法 |
| D 重复 | 每天所有 AI 场景都刻意练习 + 专项练不同类型任务 + 迁移到新场景 |

3 个月到 50 分，6 个月到 70 分，现在稳定 80 分+。

### 案例 2：逐字稿练习（伍月班主任）

目标：镜头前自然有感染力地表达。核心做法：写稿 → 录像 → 回看 → 改 → 再录。

| ABCD | 具体做法 |
|:---:|:---|
| A 套路 | 逐字稿写作模板（逐句写 + 分遍修改：内容→流畅→节奏→互动→精简） |
| B 非舒适区 | 要求自己脱稿、自然表达，对标"像在跟朋友聊天"的最终状态 |
| C 反馈 | 看录像回放（表情/眼神/语气/卡壳）+ 版本对比 + 专家（Truman）反馈 |
| D 重复 | 同一篇稿子改几十遍 + 摄像头前练 200+ 遍 + 发现问题词单独练 100 遍 |

50 遍后基本流畅，100 遍后有节奏感，200 遍后自然自信有感染力。

### 其他案例速览

3. **德州扑克**：从新手到 NL400，3 年——核心是固定套路（起手牌表 + 位置策略）+ 大量实战复盘
4. **灵感闪现**：每年一个阶段，不断升级——核心是阶段模型 + 迁移练习
5. **视频剪辑**：建子弹库 + SOP + 最佳实践池子——核心是套路文档 + 反馈池子

## 常见反模式

帮用户识别这些无效练习模式：

| 反模式 | 表现 | 对应缺陷 |
|:---|:---|:---|
| **只学不练** | 疯狂看课/读书，但不实操 | 缺 D（重复） |
| **舒适区循环** | 重复做已经擅长的事 | 缺 B（非舒适区） |
| **无脑重复** | 每次做法不同，没有框架 | 缺 A（套路） |
| **闭门造车** | 练完不看结果、不要反馈 | 缺 C（反馈） |
| **三分钟热度** | 开始很猛，两周后放弃 | 动机问题 |

完整的策略武器库见 `references/strategy-cheatsheet.md`，经典案例详解见 `references/case-studies.md`。
