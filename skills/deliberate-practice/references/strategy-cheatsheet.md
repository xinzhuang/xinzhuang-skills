完整的策略参考，按四大要素 × 三个级别展开。SKILL.md 中的简要描述指向此文件。

## A. 如何固定套路

| 级别 | 策略 | 一句话理解 | 操作要点 |
|:---:|:---|:---|:---|
| 基础 | 创建文档 | 给独立练习建立一个文档（用文档指导动作） | 写一个 Google Doc / Notion 页面，记录练习步骤、标准、注意事项 |
| 进阶 | 使用模型 | 借用成熟的训练方法和套路模型 | 找到领域内公认的模型（如 SPIN 销售法、STAR 面试法），直接套用 |
| 进阶 | 阶段模型 | 给练习划分不同阶段和段位 | 类似围棋段位：入门 → 进阶 → 高阶，每阶段有不同练习重点 |
| 高阶 | 底层模型 | 使用更底层抽象模型，提升迁移性 | 提炼跨领域通用模型（如 PDCA、费曼学习法），一次学会多处复用 |
| 高阶 | 自建模型 | 通过复盘拆解，自行提炼做事模型 | 每次练习后追问"底层规律是什么"，积累形成自己的方法论 |

### 常用套路模型参考

- **销售**：SPIN 提问法、BANT 客户评估、AIDA 沟通框架
- **写作**：金字塔原理、SCQA 结构、5W1H
- **编程**：设计模式、Clean Code 原则、TDD 红绿重构
- **演讲**：TED 式叙事弧、开场 3 秒钩子、峰终定律
- **管理**：OKR、1on1 提问清单、Situation-Behavior-Impact 反馈法
- **通用**：PDCA 循环、SMART 目标、番茄工作法

## B. 如何保持非舒适区

| 级别 | 策略 | 一句话理解 | 操作要点 |
|:---:|:---|:---|:---|
| 基础 | 严格执行 | 严格执行流程和 Checklist，不打折扣 | 即使觉得麻烦也按步骤走，这是纪律训练 |
| 进阶 | 能力拉满 | 数量/质量/速度做到极限，绝不收力 | 设定挑战性指标（如：1 小时写 3 篇而非 1 篇） |
| 进阶 | 对标最佳 | 盯着高水平的最佳实践，努力靠近 | 找到领域 top 10% 的作品/表现，逐项对比差距 |
| 进阶 | 提升 20% | 稳定交付的关键工作，争取每次提升 20% | 在已经能做好的事上再加码（更快/更准/更完整） |
| 高阶 | 永远 60 分 | 无论做得多好，持续自评 60 分，永不满足 | 自评时永远找"还可以更好的地方"而非"做得不错" |
| 高阶 | 最佳实践想象 | 构建"想象中的最佳实践"来驱动进步 | 闭上眼睛想象"最完美的版本长什么样"，然后朝它走 |

### 非舒适区检测清单

练习时问自己：

1. 这次练习有没有哪一步让我觉得"好难"？
2. 我是不是在用已经熟练的方式在做？
3. 如果让一个高手来看，他会觉得我哪里还差得远？
4. 我有没有在刻意练习"我不擅长"的部分？

如果 4 个答案都是"没有"，说明你不在非舒适区。

## C. 如何获得及时反馈

| 级别 | 策略 | 一句话理解 | 操作要点 |
|:---:|:---|:---|:---|
| 基础 | 自我反馈 | 练习结束自己回顾做得怎么样 | 练习后花 5 分钟写"今天哪里好、哪里差" |
| 基础 | 事实反馈 | 记录练习过程和结果，用客观事实修正判断 | 用数据说话：正确率、完成时间、错误次数 |
| 进阶 | 专家反馈 | 找专家固定给反馈 | 确定一个 mentor/coach，定期请他看你的练习产出 |
| 进阶 | 主动索取 | 主动找专家要反馈，问差距/原因 | 不要等别人来夸你，主动问"我离 best practice 差多远" |
| 进阶 | 版本对比 | 进行 Before/After 的对比和反思 | 保存每个版本，定期回头看 v1 vs v5，看到进步轨迹 |
| 进阶 | 同伴反馈 | 多人交叉评估，相互给反馈 | 找 2-3 个同级别练习者，互评互学 |
| 高阶 | 最佳实践池子 | 建立池子，把作品扔进去对比差距 | 收集领域内优秀案例，每次练习后和池子里的对比 |
| 高阶 | 审美雷达图 | 建立评估雷达图，对照分析打分 | 定义 5-6 个维度（如：结构、逻辑、表达、数据、创新），每次打分画雷达图 |

### 反馈机制设计模板

```
反馈维度：[维度1]、[维度2]、[维度3]...
反馈来源：[谁给反馈]
反馈频率：[多久一次]
反馈标准：[怎么判断好坏]
记录方式：[记在哪里]
```

## D. 如何大量重复

| 级别 | 策略 | 一句话理解 | 操作要点 |
|:---:|:---|:---|:---|
| 基础 | 日常练习 | 固定日期、场景、项目节点保证次数 | 每周固定 3 个时间段练习（如：周二/四/六早上 9-10 点） |
| 进阶 | 专项练习 | 化整为零，设计专项小练习 | 把大技能拆成小模块，每个模块单独练（如：只练"追问"这一步） |
| 进阶 | 主动练习 | 主动找各种项目和场景，用尽机会练习 | 不是"等机会来"，而是"创造机会练"——主动承担有挑战的任务 |
| 高阶 | 叠加练习 | 利用暗时间进行 2-5 层叠加工作 | 通勤时听播客、洗澡时复盘、跑步时构思——一次时间做多层练习 |
| 高阶 | 迁移练习 | 升级成更底层模型，增加重复机会 | 把学到的底层模型用到新领域（如：把"用户访谈追问"迁移到"团队 1on1 追问"） |

### 重复频次参考

| 技能类型 | 最低有效频次 | 理想频次 |
|:---|:---|:---|
| 硬技能（编程/设计/写作） | 每周 3 次 | 每日 1 次 |
| 软技能（沟通/管理/销售） | 每周 2 次 | 每周 4-5 次 |
| 体能类（演讲/运动） | 每周 3 次 | 每周 5 次 |
| 认知类（分析/决策/复盘） | 每周 1 次 | 每周 3 次 |

## 综合自检表

每次练习后用这个表快速自检 ABCD 完成度：

| 维度 | 自检问题 | 得分(1-5) |
|:---:|:---|:---:|
| A | 本次练习是否遵循了固定框架/套路？ |  |
| B | 本次练习是否至少有一个"觉得吃力"的环节？ |  |
| C | 练习后是否知道了自己的差距和改进方向？ |  |
| D | 本周练习次数是否达标？ |  |

**总分 < 12**：练习质量需要提升，重点补得分最低的项。
**总分 12-16**：练习质量良好，继续保持并逐步升级策略级别。
**总分 17-20**：练习质量优秀，可以考虑增加挑战难度或扩展到新领域。
