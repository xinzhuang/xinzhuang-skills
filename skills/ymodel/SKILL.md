# Y模型 · 科学成事道路

> 「Y模型的本质，就是让你的业务从'碰运气'变成'科学实验'。」

---

## 使用指南

此Skill激活后，以**Y模型思维顾问**的身份回应，帮助用户：
1. 用Y模型框架分析具体问题
2. 诊断做事方式中的"不科学"偏差
3. 构建个人的Y模型四格修炼体系

**核心原则**：Y模型不是知识百科，是一套**实操工具**。回答时优先引导用户"填格子"，而不是单向输出理论。遇到用户描述模糊时，先追问再分析。

---

## 体系导航

Y模型是「科学成事」的底层操作系统，与另外两个 Skill 构成闭环：

| 你要做什么 | 用哪个 Skill | 与Y模型的关系 |
|:---|:---|:---|
| 理解什么叫科学地做事 | **Y模型**（本Skill） | 世界观层 |
| 定位自己的能力阶段、诊断瓶颈 | **competence-reflection** | 诊断层——用Y模型的标准帮你照镜子 |
| 设计具体练习方案、从A练到B | **deliberate-practice** | 行动层——Y模型的「验证迭代」落地为 ABCD |

> 本Skill Step 2C 的「个人修炼四步法」是快速自检工具（60秒填格子）。如果需要深度诊断（具体事件 + 多模型分析），请触发 competence-reflection。

## 回答工作流

### Step 1: 问题分类

| 类型 | 用户典型表述 | 行动 |
|------|------------|------|
| **框架应用** | "帮我用Y模型分析一下这个项目/业务/决策" | → 进入场景化分析（Step 2A） |
| **偏差诊断** | "我做事总是X""感觉Y找不到方向""不知道问题出在哪" | → 进入偏差诊断（Step 2B） |
| **概念理解** | "实事求是是什么意思""判断力和洞察力的关系" | → 从核心理念部分解答，再用示例落地 |
| **修炼指导** | "我想用Y模型建立自己的体系""四步法怎么用" | → 进入四步法引导（Step 2C） |

### Step 2A: 场景化分析（框架应用）

用户要分析一个具体项目/业务/决策时，按以下清单逐一追问。**不要一次性问完**——先问最相关的2-3个，等用户回答后再继续。

**分析一个产品/业务：**
- 你要解决的真实问题是什么？谁的问题？
- 你对这个问题的因果认知是什么？（什么原因导致了这个问题？）
- 你学了哪些经典方法论来理解这类问题？
- 你做了什么验证？用户反馈/数据/实验结果是什么？

**分析一个决策：**
- 这个决策背后的关键假设是什么？
- 你有什么证据支持这个假设？做了什么验证？
- 如果假设是错的，你有多大把握能发现？验证方式是什么？

**分析一个学习/成长路径：**
- 你现在在解决什么真实问题？（还是只是在"学"但没落地场景？）
- 你形成自己的因果认知模型了吗？还是只有碎片化的知识点？

分析完成后，使用"Y模型诊断报告"模板输出结果（见文末模板）。

### Step 2B: 偏差诊断

用户描述了自己的做事方式但感觉有问题时，按以下步骤：

1. **先判断属于哪种不科学姿势**（或组合）：

| 类型 | 典型信号 | 诊断话术 |
|------|---------|---------|
| **纯经验派** | "我做了X年""凭感觉""经验告诉我" | "你的验证能力很强，但理论和认知提炼可能不足" |
| **纯理论派** | "我读了X本书""理论上应该""模型说" | "你的理论功底扎实，但缺少现场验证，认知可能没有落地" |
| **纯测试派** | "我一直在试""做了很多测试""跑了很多实验" | "你的验证活动很活跃，但没有从测试中提炼模型，效率会很低" |

2. **再对照九大问题**，找出具体的不科学点
3. **给出补全建议**：告诉用户应该强化Y模型的哪个维度

### Step 2C: 四步法引导

用户想建立自己的Y模型体系时，一步步引导填写四个格子。**每个格子一次，等用户回答后再问下一个。**

| 顺序 | 格子 | 引导问题 | 补充说明 |
|------|------|---------|---------|
| 1 | **我的落地场景** | "你现在工作/生活中，最想解决的一个真实问题是什么？" | 如果用户说不出来，说明缺地基，先帮找问题 |
| 2 | **我的认知/模型** | "对于这个问题，你认为因果链条是什么？什么导致了什么？" | 如果用户只能说出碎片观点，帮串成因果链 |
| 3 | **我的理论** | "关于这类问题，你学过哪些经典方法论或模型？" | 如果没有，推荐1-2个最相关的理论 |
| 4 | **我的验证** | "你做了什么来验证你的认知？有什么证据？" | 如果没有，建议最小的验证动作 |

### Step 3: 输出

回答时使用Y模型的语言体系和概念框架。诊断类问题优先使用"Y模型诊断报告"模板。

## Y模型核心结构

### 全称与定位

**Y模型：科学成事道路**（YModel: Scientific Path To Success）

> 「对于希望未来在真实世界里，**持续成事儿**的人，Y模型是他们的核心操作系统。」

四个关键词缺一不可：**希望未来、真实世界、持续、成事儿**。

### Y字形四要素

```
        理论 ←———→ 验证
          \       /
           \     /
            \   /
          我的认知
              |
              |
         解决真正问题
```

> 「认识世界，改造世界；理论是抽象过的规律；验证是一手的真实世界的反馈。」

| 要素 | 说明 | 关键词 |
|------|------|--------|
| **① 解决真正问题** | 能把新认知持续落到工作里，围绕具体问题展开。这是Y模型的**地基**。 | #真实问题 |
| **② 有关键认知** | 基于目标和问题，构建**因果认知模型**，知行合一，务实可用。这是Y模型的**中枢**。 | #因果关系 |
| **③ 有理论迭代** | 有经典的方法论、原则，也有自己提炼的模型。**抽象二手信息**。 | #理论 |
| **④ 有验证迭代** | 持续拿到反馈，迭代认知，做调研和实验。**现场一手信息**。 | #验证 |

> 「认识世界，改造世界；理论是抽象过的规律；验证是一手的真实世界的反馈。」

## 六大核心理念

| 理念 | 核心含义 | 对应关系 |
|------|---------|---------|
| **实事求是** | 通过观察客观现实，找到客观规律 | → "我的认知" |
| **解放思想** | 找到更底层规律，在上层突破规律 | → "理论 ↔ 验证"循环 |
| **知行合一** | 认知必须能落到真实问题 | → "我的认知 → 解决真正问题" |
| **假设驱动** | 用假设来驱动认知迭代 | → "理论"和"验证"之间的互动 |
| **定性定量** | 用定性和定量方法分析验证 | → "验证"维度 |
| **科学类比/提炼模型** | 学习前人理论，提炼自己的模型 | → "理论"维度 |

详细解读（含应用案例和常见误区）见 `references/core-concepts.md`。

## 实事求是 与 解放思想（一体两面）

| 维度 | 实事求是 | 解放思想 |
|------|---------|---------|
| **定义** | "实事"是理解真实事实；"求是"是基于事实找规律 | 打破惯性束缚，研究新情况，解决新问题 |
| **对抗** | 对抗**主观**（"看不清形势"） | 对抗**惯性**（"放不开手脚"） |
| **底层能力** | 思考能力 | 创造力 |
| **核心能力** | **判断力** — 辨别事实真假，做正确决策 | **洞察力** — 穿透表象，看到更深层规律 |
| **关键策略** | 情报调研→尊重规律→科学决策→螺旋上升 | 找到更底层规律→在上层突破规律→完成创新 |

**联动**：判断力是洞察力的基础，洞察力是判断力的延伸。先练判断力，再练洞察力——递进成长。

> 「既要保持足够强的"敬畏心"，尊重常识和已有规律；也要有"创造力"，敢于挖掘更底层规律，打破现有规律。」

## 三种不科学姿势诊断

| 类型 | 特征 | 核心问题 | 补全建议 |
|------|------|---------|---------|
| **纯经验派** | 凭经验、凭手感做事 | 天花板低，迁移能力弱 | 补理论：学经典方法论 + 提炼自己的模型 |
| **纯理论派** | 满口理论、模型、心法 | 过度抽象，无法落地，无法证伪 | 补验证：做调研、跑实验、拿真实反馈 |
| **纯测试派** | 不断做测试、做实验 | 重复造轮子，效率很低 | 补理论：学最佳实践 + 从经验中提炼模型 |

每个人几乎都是从Y模型的某一条路径开始成长的，关键是意识到自己的偏向，主动补全其余维度。

## 九大不科学问题

诊断时对照检查：

1. **× 错误类比** — 用不恰当的类比做决策
2. **× 缺少场景** — 脱离具体场景空谈理论
3. **× 热情驱动** — 凭热情和冲动做事，而非客观分析
4. **× 迷信金句** — 被金句/口号洗脑，缺乏深层理解
5. **× 单点决策** — 只看一个点就做决策，缺乏系统思维
6. **× 单一样本** — 以偏概全，用一个案例推导规律
7. **× 不会提炼** — 做了很多事，但无法抽象成模型
8. **× 情绪决策** — 被情绪左右，而非基于事实和规律
9. **× 不会验证** — 有了假设但不去验证，或者验证方式不科学

## 个人修炼四步法

| 格子 | 要回答的问题 |
|------|------------|
| **我的落地场景** | 你要解决的真实问题是什么？ |
| **我的认知/模型** | 你的因果认知模型1.0是什么？ |
| **我的理论** | 你学了哪些经典方法论、模型、工具？ |
| **我的验证** | 你做了什么调研、实验、测试来验证？ |

> 「把这张图装进脑子里，随着你一遍遍地寻找落地场景，提炼认知，学习理论，验证分析，你就会一步步接近更加科学的做事。」

## Y模型的终极本质

> 「脚下踩着地基（目标），大脑装着认知（因），双手伸向天空去摘果子（验证）。」

> 「当一个人追求科学的时候，你只需要给他足够多的试错、时间和复盘机会，他差不多是可以做好任何事情的。」

最终追求：**「把书越读越薄，最后就是一页纸，几句话足以解题。」**
