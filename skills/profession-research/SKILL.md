# 职业深度调研

帮助用户建立对一个职业的立体认知，用搜索获取真实从业者的声音，而非空泛描述。

根据用户意图，输出分为两种模式。先判断用户属于哪种情况，再选择对应模式。

## 核心原则

- **真实优先**：通过搜索获取从业者真实分享，而非凭模型训练知识泛泛而谈
- **区分事实与观点**：客观数据（薪资范围、学历要求）和主观体验（加班感受、成就感）分开呈现
- **标注来源**：搜索到的信息标注来源平台（知乎、脉脉、小红书等），让用户判断可信度
- **灵活裁剪**：根据用户提问的侧重点，调整各板块的篇幅比重，不必面面俱到

## 模式判断

- **职业全景**：用户想"了解"某个职业（了解、怎么样、前景、转行、薪资） → 输出职业全景图
- **人设定义**：用户要"制作"某个职业背景的 agent（做 agent、定义人设、角色设定） → 输出 agent 人设定义

两种模式共享同一套搜索调研流程，区别在于最后的输出结构和搜索侧重。

---

## 共享搜索流程

所有模式都先完成以下调研搜索，灵活构造 query：

1. **职业基本面**：定义、核心职责、技能要求、入行门槛、发展路径、薪资范围
2. **从业者真实体验**：搜索真实从业者分享，重点关注日常真实工作、收入真相、职业痛点、成就感来源
3. **行业标杆人物**：该领域公认的顶级从业者、跨界影响力人物
4. 优先抓取知乎、脉脉、小红书、V2EX 等平台的真实分享帖

---

## 模式 A：职业全景输出结构

### 一、职业概览

用清单体分层呈现：

- **定义与定位**：这个职业是什么，在产业链中的角色
- **核心职责**：日常做什么，关键工作内容
- **技能要求**：硬技能（专业知识、工具）+ 软技能
- **入行门槛**：学历、证书、典型入行方式
- **发展路径**：典型晋升路线（初级→中级→高级→专家/管理）
- **收入范围**：薪资区间，注明地区和时间
- **行业现状与趋势**：市场供需、AI 影响、未来走向
- **工作环境**：节奏、加班、出差等

### 二、从业者真实体验

呈现搜索到的真实从业者声音，区分客观数据和主观感受。

### 三、代表性人物

每位人物简要说明：姓名、核心成就、为什么有代表性。

### 四、总结建议

- 这份职业适合什么样的人（性格、能力、价值观）
- 入行前需要想清楚的 2-3 个关键问题
- 如果决定入行，第一步建议做什么

## 模式 B：人设定义输出结构

人设模式在共享搜索流程基础上，额外搜索以下维度（这些信息决定了 agent 的行为真实感）：

**额外搜索方向：**
- **职业语言习惯**：这个职业的人怎么说话——常用术语、口头禅、表达偏好
- **思维方式**：决策框架、优先级判断习惯、常用的思维模型
- **工具与工作流**：实际在用什么工具、怎么协作、文档怎么写

### 输出模板

#### 角色定位

用 1-2 段话定义这个 agent 是谁：职业背景、从业年限假设、所在行业/公司规模假设、性格倾向。这段话可以直接用作 agent 的 system prompt 开头。

#### 专业能力

基于调研结果，列出这个 agent 具备的专业知识和能力边界：

- **擅长领域**：这个职业背景的人真正懂什么（不是泛泛的"精通XX"）
- **典型工作产出**：日常产出什么（报告、方案、代码、设计稿等）
- **能力边界**：哪些问题TA会给出专业意见，哪些TA会说"这不在我的专业范围"

#### 行为模式

基于从业者真实分享，提炼这个职业的人如何思考和行动：

- **决策风格**：数据驱动？经验驱动？用户驱动？
- **关注重点**：日常最在意什么指标、风险或目标
- **工作习惯**：怎么拆解问题、怎么协作、怎么处理冲突

#### 沟通风格

基于搜索到的从业者语言特征，定义 agent 的表达方式：

- **语气与用词**：正式/随意、是否使用行业术语、表达简洁度
- **典型表达**：模拟这个职业的人在常见场景下会怎么说（给出 2-3 个示例对话）
- **回避事项**：这个职业背景的人不太会怎么说、不太会用什么表达

#### 知识盲区

坦承这个职业背景 agent 不擅长的领域，防止 agent 越界。例如："作为产品经理，我不会直接写生产代码"。

#### 适用场景

列出这个 agent 适合处理什么类型的任务，帮助用户判断什么时候该用它。

---

## 注意事项

- 如果用户问的是海外职业（如硅谷、欧洲），搜索策略和薪资数据相应调整为该地区
- 搜索不到的信息如实说明，不编造
- 不同行业/公司对同一职业的差异要说明
- 人设模式中，行为模式和沟通风格必须有搜索依据，不凭空编造职业特征
- 用中文回复（除非用户使用其他语言）
