# 提问的智慧 — 通用方法论

> *"如果给我一小时解决问题，我会花 55 分钟思考问题本身，5 分钟思考解决方案。"* — 爱因斯坦
>
> *"问题的表述往往比问题的解决更重要。"* — 爱因斯坦

## 核心理念

**问题本身就是核心工作。** 大多数糟糕的答案源于糟糕的问题。在向任何人提问之前，先投入这"55 分钟"——思考到问题足够锋利，答案几乎自动浮现。

**好问题是对回答者的礼物。** 它们锐化思维、暴露隐藏假设、为双方创造洞察。一个精心设计的问题会让专家 *想要* 参与。

**尊重最稀缺的资源：他人的时间。** 无论问开发者、CEO、医生还是导师——他们理解你问题的每一分钟，都是没能用来解决问题的分钟。让这个投入尽可能小、尽可能有回报。

**症状优于诊断。** 呈现你观察到的，而非你认为的原因。你的解读可能是错的；你的观察是有价值的原始材料。

## 55 分钟框架：提问前先思考

### 阶段 1：搜索（第 0-15 分钟）
**在占用任何人的时间之前，先穷尽已知信息。**

- 搜索网络、论坛、文档、书籍、过往对话
- 问自己："有人已经回答过这个问题吗？答案会在哪里？"
- 问自己："专家会用什么搜索词？"（通常和你的第一直觉不同）
- 记录你找到了什么——即使"没找到相关内容"也是有价值的信号

### 阶段 2：分解（第 15-30 分钟）
**把模糊的雾气拆解成具体、可回答的碎片。**

- 我理解上的确切缺口是什么？（不是"我不懂"——而是"我理解 X 和 Y，但 Z 说不过去"）
- 我的假设是什么？哪个假设我最不确定？
- 如果只能问一个问题，那会是什么？
- 我问的是真正的目标，还是仅仅是我卡住的那一步？

### 阶段 3：重新定义（第 30-45 分钟）
**检查你是否在问正确的问题。**

- **目标 vs 步骤**："我想做 X" → X 真的是我需要的吗？还是我执着于某一条路径？
- **XY 问题检测**：我是在问我的尝试方案 Y，而不是真正的问题 X 吗？
- **视角切换**：专家会怎么定义这个问题？他们能看到什么我看不到的？
- **逆向提问**：如果我想在这件事上失败，我会怎么做？（暴露隐藏约束）

### 阶段 4：组织（第 45-55 分钟）
**为最大清晰度和最小摩擦精心构建问题。**

- 先给上下文，再给具体请求
- 展示你已经尝试和学到的
- 提供具体例子，而非抽象描述
- 预判后续问题并提前回答

## 好问题的解剖

### 通用结构

```
1. 背景    — 我是谁，我在做什么，为什么这很重要
2. 目标    — 我最终想达成什么（真正目标，不只是卡住的那一步）
3. 我做了什么 — 研究、尝试和学到的（证明你投入了 55 分钟）
4. 缺口    — 具体的、狭窄的卡点
5. 请求    — 一个精确的、可回答的问题
```

### 反模式速查表

| 反模式 | 为什么失败 | 改进方法 |
|---|---|---|
| "你能帮我吗？" | 语义为空——是/否，没有实质 | 说明具体问题和已尝试的方法 |
| "不行" / "不工作" | 零信息量 | 描述实际发生了什么 vs 预期什么 |
| "你觉得 X 怎么样？" | 太宽泛，邀请漫谈 | 收敛到一个具体决策或缺口 |
| "急！！！" | 你的紧急 ≠ 他们的优先级；触发过滤 | 每个好问题都值得被认真对待 |
| "我只是个新手，对不起……" | 卑微浪费时间，暗示懒惰 | 清晰呈现事实；能力通过努力体现 |
| XY 问题：需要 X 却问 Y | 回答者解决了错误的问题 | 先说真正目标，再说卡住的步骤 |
| "X 可行吗？" | 是/否——不太可能是你真正需要的 | 描述你想达成的效果；问方法 |
| "给我讲讲 X" | 无边界请求——不尊重时间 | 问你现在需要的具体内容 |
| 猜测而非观察 | "我觉得是因为……"浪费时间在错误诊断上 | "我观察到的是：[证据]" |

### 精确度阶梯

把你的问题往上移：

```
Level 0:  "坏了。"                                     ← 零信息
Level 1:  "我做 Y 的时候 X 不工作。"                    ← 基本症状
Level 2:  "在环境 W 中做 Y 时，X 报错 Z。"              ← 可复现 + 证据
Level 3:  "在 W 中做 Y 时 X 报 Z。我试了 A、B、C。     ← + 已尝试 + 缩小了缺口
           A 没变化，B 产生了不同的错误 E。"
Level 4:  "X 在 W 中报 Z。试了 A/B/C。我的目标是 G      ← + 最小复现 + 目标清晰
           （不一定是 X）。这是触发它的最小用例。
           我怀疑问题在 [区域]，因为 [原因]。"
Level 5:  Level 4 + "我读了文档 D 和 E，搜索了 F 和 G， ← 完整展示了 55 分钟的投入
           找到了相关问题 H。缺口具体在 [已知] 和
           [未知] 之间。"
```

**目标：至少 Level 3。高风险请求追求 Level 5。**

## 领域适配

### 技术 / 工程
- 最小可复现测试用例
- 环境：操作系统、版本、硬件、配置
- 错误日志和会话记录
- 按时间顺序的事件序列

### 业务 / 战略
- 决策上下文：桌面上的选项、约束、时间线
- 已有数据和缺失数据
- 利益相关方和已识别的权衡
- "如果不做任何事，后果是 X"

### 研究 / 学术
- 已阅读的文献；你在研究中的位置
- 你的假设和验证方法
- 方法论可能存在的缺口
- "领域认为 X，但我观察到 Y——这是真正的缺口吗？"

### 个人 / 职业
- 你的处境、目标和约束（诚实面对）
- 已尝试的方法和结果
- 需要的具体决策或洞察
- "我在 A 和 B 之间纠结，因为 [权衡]"

### 创意 / 设计
- 受众、目的和约束
- 已尝试的方案和不足之处
- 参考案例和反例
- "我想让它感觉像 X，但目前感觉像 Y"

## 问题质量评分

按 6 个维度打分（1-5 分）：

| 维度 | 1 分（差） | 3 分（合格） | 5 分（优秀） |
|---|---|---|---|
| **研究深度** | 没有努力的迹象 | 提到一些尝试 | 精确列出尝试过和发现的 |
| **精确度** | 模糊、无焦点 | 足够具体 | 锋利；缺口边界清晰 |
| **目标清晰度** | 隐藏或混乱 | 说了要什么 | 说了真正目标；开放更好路径 |
| **简洁度** | 大量噪音 | 长度合理 | 每个字都有存在的理由 |
| **证据** | 只有观点/猜测 | 一些具体细节 | 原始观察、数据、例子 |
| **重新定义** | 困在一个视角 | 有一定自省 | 主动质疑自身假设 |

**18+/30 → 发送。低于 15 → 继续思考。**

## 发送前自检清单

在发出问题之前，快速核对：

- [ ] 我展示了搜索和研究的证据吗？
- [ ] 问题是具体可回答的，还是模糊发散的？
- [ ] 我陈述的是真正的目标，还是仅仅是卡住的步骤？（XY 问题检测）
- [ ] 去掉任何一句话后问题是否仍然完整？如果是，删掉它。
- [ ] 一个陌生人能仅凭这段文字理解我的处境吗？
- [ ] 我是否预判了最可能的后续问题并提前回答？

## 快速模板

### 问题报告（任何领域）
```
背景：[情况、环境、背景]
目标：[成功长什么样]
症状：[哪里出了问题——观察到的，不是猜的]
已尝试：[具体方法，附结果]
缺口：[我不理解/无法解决的具体问题]
请求：[一个清晰的、可回答的问题]
```

### 决策请求
```
背景：[情况和利害关系]
我看到的选择：[A、B、C 及各自的优缺点]
我的倾向：[及理由——让你挑战我的推理]
约束：[必要条件和不可接受条件]
请求：[我具体需要你提供什么]
```

### 知识缺口
```
我理解的：[表明我不是从零开始]
我困惑的地方：[具体的概念边界]
我读过/试过的：[证明那 55 分钟]
我的最佳猜测：[标明是猜测，欢迎纠正]
请求：[能帮我突破的问题]
```

### 反馈请求
```
我做的/做的：[链接、成果、描述]
我优化的目标：[清晰的成功标准]
我不确定的：[信心最低的部分]
请求：[具体反馈维度，不是"你觉得怎么样？"]
```
