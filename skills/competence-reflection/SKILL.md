# 能力阶段自我觉察 · Competence Reflection

> "知道自己不知道，是成长最快的阶段——前提是你真的知道。"

帮用户从"学了但没用上"的困惑中走出来，通过多模型交叉定位，找到真正的瓶颈，设计可执行的下一步行动。不是给你讲理论，是给你一面镜子。

---

## 体系导航

本 Skill 是「科学成事」体系的诊断层，与另外两个 Skill 构成闭环：

| 你要做什么 | 用哪个 Skill | 定位 |
|:---|:---|:---|
| 理解什么叫科学地做事 | **ymodel** | 世界观层 |
| 定位能力阶段、诊断瓶颈 | **competence-reflection**（本Skill） | 诊断层 |
| 设计具体练习方案 | **deliberate-practice** | 行动层 |

> **典型用户旅程**：ymodel 建立认知 → 本Skill 诊断瓶颈 → deliberate-practice 设计练习 → 回到本Skill Step 6 校验进步。

---

## 术语映射

本 Skill 使用心理学学术模型（达克效应、Dreyfus、Miller），与 Y模型 语言的对应关系：

| 本 Skill 术语 | Y模型术语 | 含义 |
|:---|:---|:---|
| 愚昧之巅（达克效应） | 缺少实事求是 | 不知道自己不行，缺少对真实水平的认知 |
| 绝望之谷（达克效应） | 开始实事求是 | 看到了真实差距，痛苦但正确 |
| 有意识不胜任（四阶段） | 知行合一的起点 | 知道标准但做不到，需要刻意练习 |
| Shows How → Does（Miller） | 验证迭代的关键跨越 | 模拟能做但实战做不了，是Y模型右叉（验证）的典型瓶颈 |
| 知其时然未内化 | 认知模型不够具体 | 知道该用什么但高压下本能覆盖 |

## 工作流

```
Step 0：情绪感知      —— 用户带着什么来的？先接住人，再分析问题
Step 1：事件锚定      —— 用具体事件取代抽象自我评估
Step 2：多模型定位    —— 四个模型交叉验证，按能力维度分别定位
Step 3：知行差距诊断  —— "知道"和"做到"之间卡在哪？
Step 4：瓶颈拆解      —— 第一性原理找根因
Step 5：行为改变设计  —— 阶梯式渐进路径，每次只聚焦一个阶梯
Step 6：觉察校验      —— 用可观察信号确认真实进步
```

步骤不必顺序走。Step 0 + Step 2 是前置，Step 5 是输出核心。场景速查表在文末，可直接跳转。

---

## Step 0：情绪-状态感知

用户触发这个工具时很少是"平静地做评估"，更多是刚被打低分、刚发现"学了白学"。人在情绪中时自我评估准确性大幅下降，直接上模型分析效果很差。

通过用户描述事件的方式判断状态，调整诊断基调：

| 用户信号 | 状态 | 怎么接 |
|:---|:---|:---|
| 反复强调"我学过了""我明明知道" | 防御 | 先共情，再用事实引导"看到底卡在哪一步" |
| "我就是不行""我可能不适合" | 自我否定 | 先肯定"能意识到问题就是进步"，再做定位 |
| "我不理解为什么分数这么低" | 困惑 | 重点做外部信号校准 |
| "我想提升XX能力" | 理性主动 | 直接进入诊断，效率优先 |
| "上次帮我分析过，我又做了一次" | 回访 | 跳到 Step 6 |

**原则：先接住人的情绪，再处理事的诊断。**

---

## Step 1：事件锚定

抽象的自我评估容易自欺。引导用户描述一个**具体的输出事件**：

- 发生了什么？（场景、时间、约束条件）
- 输出是什么？结果/反馈是什么？
- 预期 vs 实际的差距有多大？
- 你对这个结果的**第一反应**是什么？（暴露情绪状态和自我认知偏差）

没有具体事件时，帮用户回忆最近的真实场景，或用假设场景替代："如果让你现在去做 [相关任务]，你有信心做好吗？为什么？"

---

## Step 2：多模型交叉定位

用四个模型从不同角度定位，交叉验证。**关键：按能力维度分别定位**，不要给一个笼统的"你在第二阶段"。同一项工作包含多个能力维度，每个维度可能在不同阶段。

四个模型的图解、诊断问题、输出模板见 `references/models.md`。

核心逻辑：
- **四阶段模型**：你在哪个能力状态？（无意识不胜任 → 有意识不胜任 → 有意识胜任 → 无意识胜任）
- **达克效应**：你的信心在哪个位置？（愚昧之巅 → 绝望之谷 → 开悟之坡 → 平稳高原）。描述信心曲线，与能力曲线常常不同步
- **Dreyfus 五层**：你的决策方式是什么级别？（新手 → 高级新手 → 胜任 → 精通 → 专家）。"靠规则做对"和"凭直觉做对"是本质不同的
- **Miller 金字塔**：你的知行差距在哪层？（Knows → Knows How → Shows How → Does）。大多数瓶颈在 Shows How → Does 之间

**自我欺骗检测**：定位时主动校准 — 自评 vs 外部评分对比、"教别人"测试、"如果重来"测试、本能反应回溯。检测到欺骗时用问题引导让用户自己发现差距，不要直接指出。

---

## Step 3：知行差距诊断

区分"知"的三个层次：知其然（知道有这个方法）→ 知其所以然（知道为什么有效）→ 知其时然（知道什么时候该用哪个方法）。

详细的差距模式库和诊断方法见 `references/diagnosis.md`。

快速判断：
- "我学过了"但说不出原理 → 知识幻觉（知其然 ≠ 知其所以然）
- 没压力时能做到，一加压就崩 → 场景退化（Shows How ≠ Does）
- 知道该用A，实际用了B → 本能覆盖（知其时然未内化）
- 学了很多但实战从不用 → 学用脱节
- 一直在做已经会的事 → 舒适区陷阱

---

## Step 4：瓶颈拆解

用第一性原理找到根因。帮用户定义所在领域的核心能力环（2-4个维度），然后把反馈/失败信号映射到对应环节，信号最集中的环节就是核心瓶颈。

能力环模板和映射方法见 `references/diagnosis.md`。

---

## Step 5：行为改变设计 — 阶梯式路径

反思如果不转化为具体的行为改变，就是精神自慰。

**设计原则：**
1. 不是"继续学"而是"做不同的事" — Does 层瓶颈靠知识解决不了
2. 每次只改一件事 — 同时改三件事等于什么都没改
3. 设计硬规则 — 不是"以后注意"，而是"每次必须先做X再做Y"
4. 设计渐进路径 — 从当前阶段到目标阶段需要多个递进步骤

**阶梯式路径：**
```
阶梯 1：建立硬规则，改变行动起点（立刻能做到）
阶梯 2：在硬规则框架内提高输出质量（需要几次练习）
阶梯 3：在递增压力下稳定输出（需要刻意练习）
阶梯 4：从"按规则做对"过渡到"自己判断做对"（需要内化）
```

**当前只聚焦一个阶梯。** 帮用户判断聚焦哪一阶：
- 连硬规则都没有 → 阶梯 1
- 有硬规则但做不到 → 阶梯 1 还没完成
- 硬规则能执行但输出质量不够 → 阶梯 2
- 无压力下能做好但高压崩 → 阶梯 3
- 高压能稳定但还靠规则 → 阶梯 4

完整的阶梯示例、行为改变模板、与刻意练习的衔接方案见 `references/action-design.md`。

---

## Step 6：觉察校验

如何确认真的在进步，而不是"感觉好了一点"但实际没变？

**每次诊断后输出反思日志**，下次校验时直接对比：

```
反思日志 — [日期]
触发事件：[一句话]
阶段定位：整体 [四阶段+Miller] | 核心瓶颈维度：[哪个]
当前聚焦：阶梯 [N]：[具体行为改变]
硬规则：[一个可执行的硬规则]
验证信号：1.[行为] 2.[输出] 3.[本能]
下次校验：[建议2-4周后]
```

**进步三信号：** 行为信号（做了什么不同的事）、输出信号（质量变化）、本能信号（第一反应变化）。**不接受纯感觉信号。**

**阶段跃迁确认：** 四阶段 — 更少退回本能？Dreyfus — 按规则做对还是自己判断做对？Miller — Does 层做到了吗？达克 — 信心稳了吗？

**没有进步时：** 检查硬规则执行次数 → 瓶颈定位是否错了 → 行为改变是否太大 → 是否在非舒适区 → 练习频次够吗（两周 < 3次等于没练）

## 与刻意练习的关系

```
competence-reflection          deliberate-practice
    ↓ 定位阶段                    ↓ 设计练习
"你在哪？瓶颈是什么？"    →    "怎么练？练什么？"
    ↓ 校验进步                    ↓ 执行练习
"进步了吗？需要调整吗？"  ←    "练完了，结果如何？"
```

Step 5 完成后自动生成练习方向，作为 deliberate-practice 的输入。主动建议用户："要设计具体的刻意练习方案吗？"

## 场景速查

| 用户说的 | 问题 | 从哪开始 |
|:---|:---|:---|
| "我学了但做不到" | 知行差距 | Step 3 → 5 |
| "我比上次好了吗" | 进步校验 | Step 6（先读上次日志） |
| "不知道差在哪" | 阶段定位 | Step 1 → 2 |
| "评委/老板说我不行" | 外部反馈校准 | Step 0 → 1 → 2 → 4 |
| "练了很久没进步" | 练习方法 | Step 4 → 5 → 用 deliberate-practice |
| "高压下就崩" | Does 层瓶颈 | Step 2（Miller）→ 5（阶梯3） |
| "我觉得我挺行的" | 愚昧之巅？ | Step 2（自我欺骗检测） |
| "上次分析过，又做了一次" | 回访校验 | Step 6（读上次日志） |

## 反模式

| 反模式 | 表现 | 干预 |
|:---|:---|:---|
| 反思成瘾 | 不断分析但从不行事 | 强制 Step 5，输出具体行为改变 |
| 模型迷信 | 用标签定义自己 | 追问"那你下次会做什么不同？" |
| 舒适区反思 | 只反思不痛不痒的问题 | 用外部反馈校准 |
| 安慰剂进步 | "感觉好多了"但输出没变 | 要求行为/输出信号 |
| 一次改三件事 | 同时改判断力+表达力+洞察力 | 强制选一个，聚焦一个阶梯 |
