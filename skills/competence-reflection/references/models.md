## 四阶段能力模型

来源：Burch, 1970s。描述能力状态的简洁框架。

```
        ┌──────────────┐
  第四层 │  无意识胜任   │  不假思索就做对了
        ├──────────────┤
  第三层 │  有意识胜任   │  知道怎么做，需要专注才能做到
        ├──────────────┤
  第二层 │  有意识不胜任  │  知道标准，但做不到
        ├──────────────┤
  第一层 │  无意识不胜任   │  不知道自己不知道
        └──────────────┘
```

**诊断问题：**
- 你是否曾经认为自己"会了"，但实际输出证明并非如此？→ 可能刚从第一阶段出来
- 你能否清楚说出"好的标准是什么"？如果说不清 → 可能还在第一阶段
- 你知道标准，但高压/限时场景下做不到？→ 第二阶段（有意识不胜任）
- 你能做到，但需要高度专注、不能分心？→ 第三阶段
- 你自然而然就做对了，不觉得费力？→ 第四阶段

## 达克效应

描述信心与能力的关系。**与四阶段的区别：** 达克效应描述信心曲线，四阶段描述能力曲线。两者常常不同步，信心的崩塌先于能力的提升——这是正常的，不是退步。

```
信心
  │    ╱╲
  │   ╱  ╲  愚昧之巅
  │  ╱    ╲
  │ ╱      ╲
  │╱        ╲
  │          ╲  绝望之谷
  │           ╲___
  │               ╲___
  │                   ╲  开悟之坡
  │                    ╲
  │                     ╲________ 平稳高原
  └────────────────────────────── 能力
```

**诊断问题：**
- 你对自己的信心近期有没有经历过大幅波动？从哪到哪？
- 你是"不知道自己不行"还是"知道自己不行"还是"开始觉得自己又行了"？
- 你现在主要是沮丧（谷底）还是在稳步积累（斜坡）？

**情绪处理：** 如果用户处于绝望之谷，不要急着分析。先肯定"能意识到问题就是进步"——从愚昧之巅掉下来是成长的必经之路。

## Dreyfus 五层模型

描述技能获取的不同阶段，核心关注**决策方式**的变化。与四阶段的关系：Dreyfus 把"有意识胜任"这一层拆得更细——同样是"能做到"，"靠规则做对"（胜任）和"凭直觉做对"（精通）是本质不同的。

```
层级          核心特征                    决策方式
──────────────────────────────────────────────────────
专家          直觉判断，不依赖规则          "就是知道该这样做"
精通          全局视角，模式识别            看到整体画面，灵活应变
胜任          能处理非标准情况              分析场景，选择策略
高级新手       开始看到全局，但靠规则办事      知道有不同情况，但不知道该选哪个
新手          严格按规则，离开规则就懵        告诉我该做什么
```

**诊断问题：**
- 面对非标准场景时，你能灵活调整方法，还是只能严格按流程走？
- 你是"靠记忆套用步骤"还是"理解原理后自己判断"？
- 遇到突发状况（时间压力、信息不全），你本能的反应是什么？
- 你能根据场景变通方法，还是只有一套固定做法？

## Miller 金字塔

来源：Miller, 1990。精确诊断知行差距在哪层。大多数"学了但没效果"的瓶颈在 Shows How → Does 之间。

```
        ┌──────────┐
        │   Does    │  真实场景中能做到
        ├──────────┤
        │ Shows How │  模拟场景中能展示
        ├──────────┤
        │   Knows   │  知道怎么应用（能解释原理）
        │   How     │
        ├──────────┤
        │   Knows   │  知道事实（能背诵知识点）
        └──────────┘
```

**诊断问题：**
- 如果现在考你，你能解释这个方法论吗？→ Knows
- 如果让你教别人，你能讲清楚怎么用吗？→ Knows How
- 如果给你一个模拟场景、没有时间压力，你能做出来吗？→ Shows How
- 在真实的高压、限时、多任务场景中，你能稳定输出吗？→ Does

**知行差距定位：**
- 前两项 Yes 后面 No → 知识层面的学习还不够
- 前三项 Yes 最后一项 No → 瓶颈在 Shows How → Does，需要实战刻意练习
- 全部 Yes → 检查是否在非舒适区，可能需要升级挑战

## 交叉定位输出模板

按能力维度分别定位，不是单一标签。

```
## 能力阶段交叉定位

### 整体定位
| 模型 | 定位 | 判断依据 |
|:---|:---|:---|
| 四阶段 | [阶段] | [为什么] |
| 达克效应 | [位置] | [为什么] |
| Dreyfus | [层级] | [为什么] |
| Miller | [层级] | [为什么] |

### 按能力维度分别定位
| 能力维度 | Miller 层级 | 四阶段 | 核心差距 |
|:---|:---|:---|:---|
| [维度 1] | [层级] | [阶段] | [差距描述] |
| [维度 2] | [层级] | [阶段] | ← 最大落差，核心瓶颈 |
| [维度 3] | [层级] | [阶段] | [差距描述] |

### 综合判断
- 模型间是否一致：[分析]
- 核心瓶颈维度：[落差最大的维度]
- 自我欺骗检测：[自评 vs 外部信号是否有偏差]
```

**如果四个模型指向一致** → 定位可信度高。
**如果模型之间有矛盾** → 深入追问，往往意味着用户对某些能力的认知存在盲区。

## 自我欺骗检测

在定位过程中主动校准，而不是等用户自己发现。

| 检测方法 | 怎么做 | 信号含义 |
|:---|:---|:---|
| **自评 vs 外部评分对比** | 让用户自评能力(1-10)，对比外部反馈分数 | 自评 >> 外部 → 可能在愚昧之巅 |
| **"教别人"测试** | 让用户用自己的话解释方法论 | 解释不清 → 知识幻觉，停在 Knows 层 |
| **"如果重来"测试** | 问"同样的任务重新做，你会怎么做" | 答案和上次一样 → 可能还没找到真正问题 |
| **本能反应回溯** | 问"高压场景下你做的第一个动作是什么" | 第一动作和方法论矛盾 → 本能覆盖 |

**重要：** 检测到自我欺骗时，不要直接指出（"你在自欺"会触发防御）。用问题引导："你说你理解这个方法，那如果场景变成X，你会怎么调整？" — 让用户自己发现差距。

## 模型间映射关系

| 四阶段 | 达克效应 | Dreyfus | Miller | 共同特征 |
|:---|:---|:---|:---|:---|
| 无意识不胜任 | 愚昧之巅 | 新手 | Knows | 不知道自己不行；自信但盲目 |
| → 过渡 → | 绝望之谷 | 高级新手 | Knows How | 发现天高地厚；信心崩塌但能力在积累 |
| 有意识不胜任 | 谷底附近 | 高级新手→胜任 | Shows How | 知道标准但做不到；成长最快也最痛 |
| 有意识胜任 | 开悟之坡 | 胜任 | Shows How→Does | 能做到但需专注；信心稳步回升 |
| 无意识胜任 | 平稳高原 | 精通/专家 | Does | 内化了；直觉判断；稳定输出 |

### 什么时候用哪个模型

| 场景 | 首选 | 辅助 |
|:---|:---|:---|
| "我学了但做不到" | Miller（精确诊断知行差距） | 四阶段（确认阶段） |
| "我为什么这么沮丧" | 达克效应（解释信心曲线） | 四阶段（确认能力在进步） |
| "我到底是新手还是高手" | Dreyfus（精细分层） | Miller（用行为验证） |
| "怎么设计下一步练习" | 四阶段 + Dreyfus（确定起点） | Miller（确定目标层级） |
