## 知的三个层次

```
知其然   "知道有这个方法"       → 看了书/上了课
知其所以然 "知道为什么这个方法有效"  → 理解了原理
知其时然   "知道什么时候该用哪个方法" → 实战判断力
```

**诊断方式：**
- 选一个用户声称"学过"的方法论
- 问：你能用一句话说清楚这个方法为什么有效吗？（测知其所以然）
- 问：如果在场景A你会用方法1还是方法2？为什么？（测知其时然）
- 如果答不上来 → 用户可能停留在"知其然"，自以为"学会了"

**类比：** 像学游泳——看过自由泳教程，也理解为什么换气要转头，但一跳进水里还是本能地抬头换气。知其所以然了，但知其时然（该转头换气的时刻）还没内化。

## 知行差距模式库

| 模式 | 表现 | 根因 | 验证方法 | 解决方向 |
|:---|:---|:---|:---|:---|
| **知识幻觉** | "我学过了"但说不出原理 | 知其然 ≠ 知其所以然 | 让用户解释"为什么这个方法有效" | 重新学，直到能教别人 |
| **场景退化** | 没压力时能做到，一加压就崩 | Shows How ≠ Does | 对比有/无压力场景的输出 | 在递增压力下练习 |
| **本能覆盖** | 知道该用A，实际用了B | 知其时然未内化 | 回忆高压场景的第一反应 | 设计"暂停-切换"硬规则 |
| **学用脱节** | 学了很多但实战从不用 | 只有输入没有练习 | 数一下最近10次实战用了几次 | 每次实战前强制选一个方法使用 |
| **舒适区陷阱** | 一直在做已经会的事 | 不在非舒适区 | 问"上次练习有没有觉得吃力的环节" | 对标最佳实践 |
| **选择困难** | 学了很多方法但不知道该用哪个 | 缺少场景-方法映射 | 给一个场景问"你会用哪个方法" | 建立场景-方法映射表 |

## 本能覆盖深度分析

"本能覆盖"是最隐蔽也最常见的瓶颈。理智知道该用方法A，高压下本能反应用了方法B。

**常见本能覆盖模式：**

| 理智知道该做的 | 本能实际做的 | 典型场景 |
|:---|:---|:---|
| 先定义问题再动手 | 直接开始执行 | 限时任务 |
| 呈现事实而非给建议 | 急于给解决方案 | 调研/咨询 |
| 先听再说 | 急于表达自己的观点 | 会议/讨论 |
| 结构化思考 | 随机发散 | 复杂问题分析 |
| 问"为什么" | 问"怎么做" | 需求分析 |

**如何打破本能覆盖：** 不是靠"记住"，而是靠"硬规则"——在行动前加一个强制步骤，让理智先于本能。例如："接到任务后前10分钟只能填受众定义表，填完之前不打开PPT。"

## 能力环分析

任何专业能力可以拆解为核心能力环（2-4个维度），每个维度可以独立定位阶段。

核心逻辑：
1. 定义所在领域的能力环
2. 把外部反馈/失败信号映射到各维度
3. 信号最集中的维度 = 核心瓶颈

### 能力环模板库

#### 调研洞察
```
洞察力 ────→ 从信息中看到别人看不到的东西
表达力 ────→ 把看到的东西让人看懂并被打动
判断力 ────→ 在约束条件下做正确的取舍
```

#### 产品设计
```
用户同理心 ──→ 理解用户真正的痛点和需求
抽象建模 ───→ 把复杂需求变成清晰的产品结构
沟通说服 ───→ 让各方理解和认同设计方案
```

#### 技术架构
```
系统思维 ───→ 理解全局约束和模块间关系
权衡决策 ───→ 在多个方案中做出正确取舍
表达沟通 ───→ 把技术决策讲清楚让团队理解
```

#### 管理
```
识人 ────→ 看到每个人的优势和局限
排序 ────→ 在众多事项中做出正确优先级判断
反馈 ────→ 让人听到真相并愿意改变
```

#### 销售
```
洞察 ────→ 理解客户的真实购买动机
提问 ────→ 用问题引导客户自己发现问题
表达 ────→ 把价值说清楚让客户被打动
```

#### 演讲表达
```
内容结构 ──→ 把复杂想法组织成清晰的故事线
情绪感染 ──→ 让听众被打动而不只是听懂
临场应变 ──→ 面对突发情况灵活调整
```

## 瓶颈映射方法

1. **收集信号：** 列出所有外部反馈（评委评语、绩效评分、项目结果、用户评价）
2. **逐条归类：** 把每条反馈映射到能力环的哪个维度
3. **统计集中度：** 哪个维度信号最集中 → 核心瓶颈
4. **交叉验证：** 用 Step 2 的多模型定位结果验证 — 模型定位的最低维度和反馈最集中的维度是否一致

### 典型瓶颈模式

| 瓶颈类型 | 特征 | 解决方向 |
|:---|:---|:---|
| **判断力瓶颈** | 知道方法但关键时刻做错取舍 | 练"先定义后执行"的硬规则 |
| **洞察力瓶颈** | 看不到关键信息，抓不住重点 | 练视角转换、用户思维 |
| **表达力瓶颈** | 有好内容但呈现不出来 | 练结构化表达、对标优秀范例 |
| **本能覆盖** | 高压下退回旧习惯 | 练"暂停-切换"机制 |

## 自我诊断问卷

### 快速定位（5 分钟）

对目标能力，诚实回答：

**1. 你能说清楚"做好这件事"的标准吗？**
- a) 说不清 → 无意识不胜任
- b) 能说清但做不到 → 有意识不胜任
- c) 能做到但需要很专注 → 有意识胜任
- d) 不用想就做对了 → 无意识胜任

**2. 面对这个能力相关的任务，你的本能反应是什么？**
- a) "这很简单" → 检查是否在愚昧之巅
- b) "好多要学的" → 绝望之谷
- c) "我知道该怎么做但要集中注意力" → 有意识胜任
- d) "上手就干" → 检查是无意识胜任还是本能覆盖

**3. 如果给你一个高压限时场景，你会：**
- a) 大脑一片空白 → 新手/高级新手
- b) 想套用方法但用不好 → 高级新手/胜任
- c) 能用方法但输出质量不稳定 → 胜任
- d) 自然而然做出正确反应 → 精通/专家

**4. 你最近一次的"失败输出"是什么时候？**
- a) 想不起来 → 可能在无意识不胜任
- b) 很近，而且我知道哪里做错了 → 有意识不胜任（好消息！）
- c) 有但不算严重 → 有意识胜任
- d) 极少 → 无意识胜任或没在挑战非舒适区

### 知行差距深测（10 分钟）

| 问题 | 是 | 部分 | 否 | 对应层级 |
|:---|:---|:---|:---|:---|
| 你能说出这个方法论的 3 个核心要点吗？ | | | | Knows |
| 你能用一句话解释为什么这个方法有效吗？ | | | | Knows How |
| 你能教一个新手使用这个方法吗？ | | | | Knows How |
| 没有压力的模拟场景中，你能正确使用吗？ | | | | Shows How |
| 有明确时间压力的场景中，你能正确使用吗？ | | | | Does |
| 高压、多任务、信息不全的场景中，你能稳定输出吗？ | | | | Does |
| 你能根据不同场景灵活调整方法吗？ | | | | Dreyfus 胜任+ |

**得分解读：**
- 前两项 Yes 后面 No → 知其然不知其所以然
- 前四项 Yes 后面 No → Shows How 层，需要向 Does 过渡
- 前五项 Yes 最后一项 No → Does 层但不稳定
- 全部 Yes → 检查是否在非舒适区
