## 设计原则

1. **不是"继续学"而是"做不同的事"** —— 如果瓶颈在 Does 层，再学知识不会有用
2. **每次只改一件事** —— 人的注意力有限，同时改三件事等于什么都没改
3. **设计硬规则** —— 不是"以后注意"，而是"每次必须先做X再做Y"
4. **设置可观察的信号** —— 你怎么知道自己在进步？
5. **设计渐进路径** —— 从当前阶段到目标阶段需要多个递进步骤

## 阶梯式改变路径

从"有意识不胜任"到"有意识胜任"，通常需要经过 3-5 个递进的行为改变。

```
阶梯 1（立刻能做到的）：建立硬规则，改变行动起点
    ↓
阶梯 2（需要几次练习的）：在硬规则框架内，提高输出质量
    ↓
阶梯 3（需要刻意练习的）：在递增压力下稳定输出
    ↓
阶梯 4（需要内化的）：从"按规则做对"过渡到"自己判断做对"
```

**每阶的完成条件：** 不靠感觉，靠可观察的信号。只有当一个阶梯的验证信号稳定达成 3 次以上，才进入下一阶梯。

---

## 阶梯示例：调研洞察能力

| 阶梯 | 行为改变 | 触发条件 | 验证信号 |
|:---|:---|:---|:---|
| 1 | 接到任务后前10分钟只填"受众定义表"，不碰内容 | 每次接到调研任务 | 表上有内容，且在开始处理内容之前完成 |
| 2 | 输出中删除所有"建议"，只保留事实和洞察 | 每次提交前自检 | 最终PPT中建议占比 < 10% |
| 3 | 在限时场景中稳定执行阶梯1+2 | 3小时模拟练习 | 限时场景输出评分 > 70 |
| 4 | 第一反应从"给建议"变成"追问事实" | 真实工作场景 | 本能反应发生变化（他人可观察） |

### 阶梯示例：演讲表达能力

| 阶梯 | 行为改变 | 触发条件 | 验证信号 |
|:---|:---|:---|:---|
| 1 | 每次演讲前写逐字稿，至少改3遍 | 每次需要演讲/分享 | 有逐字稿，且改过3遍以上 |
| 2 | 录像回看，每次记录3个改进点 | 每次练习后 | 有回看记录和改进清单 |
| 3 | 不看稿脱稿表达，录像中卡壳 < 3次 | 模拟练习 | 录像中连续流畅表达 > 5分钟 |
| 4 | 临场能根据听众反应灵活调整内容 | 真实演讲 | 听众互动明显增加，不再照稿念 |

## 阶梯选择指南

帮用户判断当前应该聚焦哪一阶梯：

| 当前状态 | 聚焦阶梯 | 判断依据 |
|:---|:---|:---|
| 连硬规则都没有 | 阶梯 1 | 没有固定的执行前步骤 |
| 有硬规则但经常做不到 | 阶梯 1（未完成） | 验证信号未稳定达成 |
| 硬规则能执行但输出质量不够 | 阶梯 2 | 硬规则执行了但外部反馈仍不好 |
| 无压力下能做好但高压崩 | 阶梯 3 | 非限时场景OK，限时场景退化 |
| 高压能稳定但还靠规则 | 阶梯 4 | 能做到但需要高度专注，不够灵活 |

## 行为改变模板

```
## 行为改变计划

### 核心瓶颈
[一句话描述真正的瓶颈]

### 当前聚焦阶梯
[阶梯 N]

### 一个行为改变
旧行为：[具体描述]
新行为：[具体描述]
触发条件：[什么时候执行新行为]

### 硬规则
[一个不可违反的执行规则]
格式：做X之前，必须先做Y

### 验证信号
1. 行为信号：[你做了什么不同的事]
2. 输出信号：[输出质量变化]
3. 本能信号：[本能反应变化]

### 对标练习
[找一份优秀的输出做对标，具体说明怎么对比]
```

## 行为改变颗粒度

好的行为改变是**动作级别**的，不是**态度级别**的：

| 层级 | 例子 | 是否有效 |
|:---|:---|:---|
| 态度 | "我要更注意方法" | 无效 |
| 方向 | "我要先定义受众再开始" | 方向对但不具体 |
| 动作 | "接到任务后前10分钟只做受众定义表" | 有效 |
| 验证 | "受众定义表填完之前不打开PPT" | 最有效 |

**判断标准：** 如果你的行为改变可以被另一个人直接观察和验证，就是动作级别的。如果只有你自己知道"有没有在注意"，就是态度级别的，需要继续往下拆。

## 与刻意练习的衔接

完成行为改变设计后，自动生成练习方向，作为 deliberate-practice 技能的输入。

```
## 练习方向建议（传入 deliberate-practice）

### 目标能力
[核心瓶颈对应的能力维度]

### 当前阶段
[Miller 层级 + 四阶段]

### 目标阶段
[下一个 Miller 层级]

### 具体练习方向
[基于阶梯路径中当前聚焦的阶梯]

### 需要的 ABCD 诊断
- A（套路）：用户是否有固定的执行框架？
- B（非舒适区）：当前阶梯是否在非舒适区？
- C（反馈）：反馈来源和频次建议
- D（重复）：建议的练习频次
```

### 衔接时机

当用户完成以下步骤时，主动建议进入练习阶段：
1. ✅ 阶段定位完成（Step 2）
2. ✅ 瓶颈诊断完成（Step 4）
3. ✅ 行为改变设计完成（Step 5）

主动话术："你已经完成了阶段定位和瓶颈诊断，也设计了行为改变方案。要设计具体的刻意练习方案吗？我可以帮你做 ABCD 诊断和练习计划。"

## 阶段跃迁标志

如何判断自己从一个阶段进入了下一个阶段。

### 从"无意识不胜任"到"有意识不胜任"

| 标志 | 说明 |
|:---|:---|
| 开始对自己的输出不满意 | 之前觉得"还行"，现在觉得"不够好" |
| 能看懂优秀作品的"好" | 之前看不出差距，现在能看到了 |
| 开始感到沮丧 | 进步信号，不是退步 |
| 外部反馈变具体了 | 之前只有"不错/不行"，现在能指出具体问题 |

### 从"有意识不胜任"到"有意识胜任"

| 标志 | 说明 |
|:---|:---|
| 模拟场景中能做到了 | 没压力时能输出合格的结果 |
| 能稳定地使用方法论 | 不再随机发挥，有稳定的执行框架 |
| 反馈从"方向不对"变成"细节可以更好" | 大问题解决了，剩下的是打磨 |
| 信心开始回升 | 不是膨胀，而是"我知道我能做到"的踏实感 |

### 从"有意识胜任"到"无意识胜任"

| 标志 | 说明 |
|:---|:---|
| 不再需要检查清单 | 方法论已内化为本能 |
| 高压下能稳定输出 | 不再因为压力退回旧习惯 |
| 能灵活调整方法 | 不是机械套用，而是根据场景变通 |
| 开始能教别人了 | 教学是最高级的内化 |
| 能直觉判断"这个不对" | 不需要分析就知道问题在哪 |

### 关键提醒

阶段跃迁不是线性的。你可能在某个能力维度是有意识胜任，在另一个维度是有意识不胜任。不要用一个标签定义整个人——按能力维度分别定位，分别设计提升路径。
